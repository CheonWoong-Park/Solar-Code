# $repo-map skill

Explore the repository and produce a comprehensive map covering:

1. **Directory Structure** — annotated tree of key directories
2. **Entry Points** — main binaries, CLI commands, API handlers
3. **Core Modules** — what each major module does
4. **Data Models** — key types and schemas
5. **External Dependencies** — major libraries and why they're used
6. **Testing Strategy** — test locations and patterns
7. **Build System** — how to build, run, and test

Save to `.oms/plans/repo-map.md`.
